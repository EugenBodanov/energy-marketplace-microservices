# Dev Terraform

This root owns the dev runtime:

- VPC, subnet, Cloud NAT, and private service networking
- Cloud SQL for PostgreSQL with private IP
- RabbitMQ on a private Compute Engine VM with a persistent data disk
- Secret Manager runtime values
- Cloud Run services and invoker IAM

Run `infra/terraform/bootstrap` first. Bootstrap enables APIs, creates Artifact Registry, configures GitHub Workload Identity, and creates the runtime service accounts expected by this root.

The deploy workflow writes `ci.auto.tfvars` with the image URIs for the current commit, then runs `terraform plan` and `terraform apply` here.

RabbitMQ is not stopped by Terraform on a schedule. Use the manual GitHub Actions workflows `start-rabbitmq-vm` and `stop-rabbitmq-vm` when you want to control VM cost.

Runtime passwords are generated by Terraform and written to Secret Manager. They are also present in Terraform state, so keep the state bucket private and versioned.

By default, Cloud Run services are public because the current service-to-service HTTP clients do not mint Google ID tokens. To make only the API gateway public later, set `require_authenticated_invocation = true` and update callers to send ID tokens to private Cloud Run services.
